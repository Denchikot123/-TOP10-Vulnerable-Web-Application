# Simple Backend JAVA

## Отчет

Я запустил первую версию бэкенда. Главная цель — скорость и минимализм. Система управляет пользователями и их личными заметками. Я выбрал **Spring Boot**, так как он позволяет быстро поднять REST-архитектуру, и H2, чтобы не возиться с установкой тяжелых БД на этапе разработки.

## Архитектурные решения
 - Стек: Java 17, Spring Web, JDBC.
 - Доступ к данным: Использован `JdbcTemplate`. Это надежный стандартный инструмент Spring, который позволяет писать чистые SQL-запросы без лишней магии ORM.

## Безопасность:

 - Эндпоинт `/api/profile` требует id. Я считаю это безопасным, так как ID — это уникальный идентификатор, который система выдает пользователю.
 - Секретные данные *(secret_note)* отделены от публичных данных в запросе списка пользователей.
 - База данных H2 работает в памяти, что исключает кражу физического файла БД.

## Реализация API
Я спроектировал два ключевых эндпоинта для масштабируемости:
 - **GET /api/users/list:** Возвращает список всех зарегистрированных имен. Полезно для поиска друзей или коллег.
 - **GET /api/profile?id={id}:** Позволяет получить детальную информацию. Я передаю id прямо в запросе, чтобы фронтенд мог легко подставлять его из URL или локального хранилища.

**Пример запроса:** `http://localhost:8080/api/profile?id=1`

**Ожидаемый ответ:** JSON с именем и секретной заметкой.
