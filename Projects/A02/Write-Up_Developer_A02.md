# Simple Backend JAVA

## Отчет

После первой [неудачной](../A01/Write-Up_Hacker_A01.md) демонстрации первой версии кода пришлось вносить правки и расширять функционал проекта.

## Архитектурные решения
 - Регистрация/вход: Были добавлены классы для регистрации/входа пользователя, в котором выдаются уникальные **токены** сессии
 - Исключение лишнего: Были убраны такие эндпоинты как /api/users/list, который выдает информацию о ID и другой личной информации каждого пользователя из базы

## Безопасность:
 - Метод выдачи токенов является максимально безопасным для пользователя, так как исключает возможность использовать перебор ID для выдачи чужой информации. Сессия связана с уникальным ID зарегестрированного пользователя, так что при вводе чужого ID сервер откажет в выдачи информации из за невалидного токена.

## Реализация API
Я спроектировал два ключевых эндпоинта:
 - **POST /api/auth/register:** Принимает JSON тело с параметрами: "username", "password", "secret_note". Регистрирует пользователя с данными в базу данных, выдает уникальный ID и токен сессии.
 - **POST /api/auth/login** Принимает JSON тело с параметрами: "username", "password". Сверяется информация из тела JSON с базой данных для выдачи его уникального токена сессии.
