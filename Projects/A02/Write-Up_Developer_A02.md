# Simple Backend JAVA

## Отчет

После первой [неудачной](../A01/Write-Up_Hacker_A01.md) демонстрации первой версии кода пришлось вносить правки и расширять функционал проекта.

## Архитектурные решения
 - Регистрация/вход: Были добавлены классы для регистрации/входа пользователя, в котором выдаются уникальные **токены** сессии
 - Исключение лишнего: Были убраны такие эндпоинты как /api/users/list, который выдает информацию о ID и другой личной информации каждого пользователя из базы

## Безопасность:
 - Метод выдачи токенов является максимально безопасным, так как полностью устраняет уязвимость IDOR / BOLA (перебор ID) для получения чужих данных. Доступ к сессии жестко связан с валидацией уникального токена

## Реализация API
Я спроектировал два ключевых эндпоинта:
 - **POST /api/auth/register:** Принимает JSON тело с параметрами: "username", "password", "secret_note". Регистрирует пользователя в базе данных и возвращает уникальный ID вместе с токеном сессии.
 - **POST /api/auth/login** Принимает JSON тело с параметрами: "username", "password". Аутентифицирует пользователя и генерирует уникальный токен сессии при успешном совпадении данных.
